# admin

## 目的
管理超级管理员工作区与统计追溯入口，包括账号、角色、权限、基础资料、日志、系统配置、异常审核和消息中心。

## 模块概述
- **职责:** 平台级账号权限管理、系统配置维护、日志审计、消息查看、报表入口、异常审核
- **状态:** 🚧开发中
- **最后更新:** 2026-03-28

## 规范
- 用户、角色、权限、配置变更必须写入操作日志。
- 超级管理员接口应返回可用于前端动态渲染的菜单和权限信息。

## 最新变更
- 基础资料中的商品管理与供应商管理已具备前端可操作维护能力，支持新增、编辑、停用与条件过滤。
- 管理端直达路由为 `/catalog/products` 与 `/supplier/list`。

- 账号管理现已支持完整的列表、详情、新增、编辑和启停用，用户可在管理员/采购/销售/库管角色间切换。
- 角色权限页改为按角色维护权限，支持回显已分配权限并重新保存。

- 日志管理现可查看用户删除及关键业务动作的统一操作日志。

- 管理员工作区现统一承担采购/销售审批确认职责，普通管理员与超级管理员可审核采购单、销售单并参与到货/入库/出库确认。

- 当前初始化账号包含 admin（SUPER_ADMIN）与 manager（ADMIN），管理员角色现拆分为超级管理员与普通管理员两级。
- 普通管理员不能把用户提升为管理员类角色；只有超级管理员可以分配 SUPER_ADMIN / ADMIN。
- 普通管理员对管理员类账号仅可查看不可操作，编辑/停用/删除均保留给超级管理员。


## 本次更新（2026-03-28）
- 新增客户管理页与仓库管理页，基础资料从“只读概览”扩展为“概览 + 独立维护页”。
- 仓库支持详情查询、编辑与启停用；客户支持筛选、详情、编辑与停用。
