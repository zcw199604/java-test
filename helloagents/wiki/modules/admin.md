# admin

## 目的
管理账号权限、基础资料、日志、系统配置、异常审核以及报表与追溯入口。

## 模块概述
- **职责:** 平台级账号权限管理、基础资料维护、日志审计、系统配置、异常审核、消息与报表入口
- **状态:** ✅可用
- **最后更新:** 2026-03-29

## 规范
- 用户、角色、权限、配置变更必须写入操作日志
- 管理员类账号的创建、编辑、停用、删除需遵守角色分级限制
- 管理端页面权限以后端返回权限与前端映射共同决定

## 当前实现
- 账号管理已支持列表、详情、新增、编辑、启停用、删除与数据范围维护
- 角色权限页支持按角色回显并保存权限树
- 基础资料覆盖商品、供应商、客户、仓库
- 管理端承担采购 / 销售审批确认和异常单据审核入口
- 日志页支持查看登录日志与操作日志

## 最新变更
- 管理员角色已拆分为 `SUPER_ADMIN` 与 `ADMIN`
- 普通管理员不能分配、修改、停用或删除管理员类账号
- 客户与仓库已补齐详情、编辑、启停用能力
- 日志、角色、权限名称已优先展示中文，减少编码暴露
