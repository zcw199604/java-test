# Changelog

## [Unreleased]

### 新增
- 补全 schema.sql 中 13 张缺失表定义（验证码、会话、密码重置、权限、角色权限、系统配置、仓库、用户数据范围、登录日志、操作日志、追溯记录、异常单据、消息）
- 新增 bulletins 表用于销售信息发布功能
- 新增 purchase_orders/sales_orders 审核/取消字段（audited_by, audited_at, audit_remark, cancel_reason）
- 补全 seed.sql 中 30 条权限定义和四角色权限映射初始数据
- 新增采购/销售订单审核（APPROVED/REJECTED）和取消（CANCELLED）完整流程
- 新增采购/销售/库存 Excel 批量导入接口（POST /api/purchases/import, /api/sales/import, /api/inventories/import）
- 新增消息自动推送机制：库存预警自动通知库管人员、审核结果自动通知订单创建者
- 新增销售信息发布模块（BulletinController/BulletinService + SaleBulletinView 前端页面）
- 新增异常单据审核通过/驳回接口（POST /api/reports/abnormal-docs/{id}/audit）
- 新增追溯记录自动写入：采购/销售各 4 个业务节点自动写入 trace_records

### 新增
- 为商品管理与供应商管理页面补齐新增、编辑、停用操作，以及关键字/状态/品类过滤能力
- 补齐账号管理页面的新增、编辑、启停用、角色切换与数据范围维护能力
- 为角色权限页面补齐权限树回显与稳定保存能力，并在初始化数据中补充权限与角色权限映射

### 修复
- 修复商品管理与供应商管理页面缺少动态路由和 SPA 刷新回退的问题，`/catalog/products` 与 `/supplier/list` 现可直接访问
- 修复商品/供应商列表仅能全量查询的问题，后端接口现支持按关键字、状态（商品额外支持品类）过滤
- 修复账号管理后端接口与数据库结构不一致的问题，用户列表详情、新增、编辑与启停用现已可用
- 修复角色权限保存仅提交权限数组导致角色元数据可能被清空的问题
- 修复库存调拨不实际转移库存数量的问题，调拨现在会扣减库存并记录正确的变更流水
- 修复采购到货前置条件从 CREATED 改为 APPROVED，销售出库前置条件从 CREATED 改为 APPROVED

### 变更
- 补充 README 与知识库中的标准启动命令，统一前端先构建、后端单入口托管页面与 API 的运行口径
- 明确项目默认启动方式为后端统一托管前端静态页面：前端构建产物发布到 `backend/src/main/resources/static/`，验收与冒烟默认不单独启动 Vite dev server
- 合并主分支后完成前端接口适配：登录验证码、消息中心、系统配置、角色权限、日志、追溯、异常页与 Excel 导出说明改为对接当前后端真实接口
- 执行 `npm install`、`npm run type-check` 与 `npm run build`，确认前端构建通过并重新生成 `backend/src/main/resources/static/` 产物
- 按任务书方向将后端认证与权限扩展为 Apache Shiro + RBAC 基础实现，新增验证码、忘记密码、重置密码、会话表与权限表
- 补齐系统管理能力：用户详情/修改、角色权限、系统配置、个人中心、修改密码、仓库管理
- 新增登录日志、操作日志、站内消息、追溯记录、异常单据与预警历史相关后端能力
- 扩展采购模块：采购需求、采购单审核/取消/跟踪/分析、Excel 导入导出
- 扩展销售模块：销售信息发布、销售单审核/取消、应收状态、Excel 导入导出
- 扩展库存模块：真实调拨、盘点盈亏上报、预警记录、Excel 导入导出
- 扩展报表模块：采销存汇总、合规追溯、异常单据、联动分析与 Excel 导出
- 补充面向任务书流程的后端集成测试样例
- 新增前端第二轮优化：引入 TypeScript 核心基础设施、`vue-tsc` 类型校验与无权限提示页
- 优化前端构建拆包与图表懒加载，降低主入口包体压力
- 增强按钮级权限控制、登录错误提示、驾驶舱加载/空态与表格加载态
- 重构前端页面为 Element Plus 后台工作台风格，新增登录页、驾驶舱、个人中心与暗黑模式
- 新增动态路由与权限菜单机制，按角色加载采购、销售、库存、管理员和统计追溯页面
- 新增统一表格/图表/导出组件，补齐采购、销售、库存、管理员、经营大屏与合规追溯页面
- 修正文档与代码现状不一致的问题，更新 README、项目技术约定与知识库
- 将 `/api/reports/export` 从 CSV 扩展为 Excel 导出能力，前端仍需按后端真实输出联调
- 修复报表中心和商品列表的前后端字段/接口使用，统一以后端真实返回为准
- 将采购流程拆分为“到货”和“入库”两个动作，新增 `RECEIVED` 状态与到货时间
- 将首页概览统计调整为数据库实时统计值，移除硬编码数据
