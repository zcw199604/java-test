# 技术方案

## 方案
1. 在采购、销售、库存服务中补充 `auditService.logOperation(...)` 调用，统一沉淀到 `operation_logs`。
2. 在用户管理后端新增 `DELETE /api/users/{id}`，删除用户前清理其会话与数据范围，再删除用户记录并写审计日志。
3. 前端账号管理页增加删除按钮与对应 API 封装。
4. 回归验证登录日志、用户管理日志、采购/销售/库存操作日志。

## 风险规避
- 删除用户可能受关联数据影响：优先清理 `user_data_scopes` 与 `user_sessions`，如仍有关联则保留错误信息提示。
- 审批/入库/出库动作日志重复：仅在业务状态更新成功后写入一次操作日志。
