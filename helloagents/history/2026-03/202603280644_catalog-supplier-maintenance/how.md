# 技术方案

## 总体方案
1. 先用真实接口做商品/供应商 CRUD 冒烟测试，确认现状。
2. 后端在商品、供应商列表接口增加查询参数支持：
   - 商品：keyword、status、category
   - 供应商：keyword、status
3. 前端补充：
   - API 封装：列表过滤、创建、更新、停用
   - 页面能力：查询栏、新增/编辑弹窗、停用按钮、刷新反馈
   - 路由注册：补齐商品管理、供应商管理页面入口
4. 重新构建前端静态资源并通过 API + 页面进行回归验证。

## 风险与规避
- 风险：动态 SQL 参数顺序错误导致查询异常
  - 规避：使用参数列表顺序拼接并用真实请求验证
- 风险：前端路由未注册导致页面不可达
  - 规避：补齐 route-map 并在构建后访问页面
- 风险：后端运行依赖 JDK 而非 JRE
  - 规避：继续使用 /mnt/dbz/jdk-21.0.4 启动与验证
