# 技术方案: 前端页面重构与后台工作台升级

## 方案概述
本次重构以前端为主，保留现有后端 API，重点升级为“企业后台工作台”风格：使用 Element Plus 统一交互组件、Pinia 管理登录态与菜单状态、Vue Router 根据角色权限动态挂载业务页面，同时补齐采购/销售/库存/管理/报表等模块化页面。

## 关键设计

### 1. 权限与路由
- 引入 `route-map.js` 维护常驻路由与业务异步路由。
- 登录后通过 `/api/auth/profile` 初始化用户资料与角色。
- 基于角色推导权限集，按权限过滤页面并构建侧边栏菜单组。
- 对无权限页面在路由守卫中统一拦截并回退到驾驶舱。

### 2. 主布局与主题
- 使用 `el-container` 构建左侧菜单 + 顶部导航 + 主内容区布局。
- 主题采用深绿 + 金色，统一抽取到 `assets/main.css` 中的变量。
- 提供暗黑模式开关与移动端 Drawer 菜单，提升可用性。

### 3. 页面体系
- 登录页改为居中卡片 + 验证码 + 记住我风格。
- 工作台加入 KPI 卡片、趋势图、预警消息、操作日志与快捷入口。
- 按模块新增采购、销售、库存、管理员、报表追溯等页面。
- 列表统一为 Element Plus 表格，导出统一为 xlsx。

### 4. 数据与复用
- 尽量复用已有 API：采购、销售、库存、报表、用户、角色等接口。
- 对后端尚未提供的管理类能力，以前端交互骨架和演示数据承接，避免虚构后端契约。
- 抽取通用组件：`PageSection`、`KpiCard`、`AppTable`、`AppChart`、`MenuTree`。

## 风险与规避
- **后端接口不足:** 对未覆盖页面使用只读展示或本地演示数据，避免伪造不存在的写接口。
- **前端包体增大:** 通过路由懒加载拆分主要页面；构建警告可接受但需记录。
- **权限信息不完整:** 使用“后端权限优先，角色兜底推导”的方式保证界面可访问。
