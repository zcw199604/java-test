# 变更提案: 烟草采销存协同管理平台初始化

## 需求背景
项目仓库当前仅包含课题说明文档，尚无可运行代码结构。根据任务书，系统需覆盖采购、库存、销售、管理员等角色能力，并形成后续可扩展的毕业设计实现基础。用户额外明确要求前端必须使用 Vue，而不是 JSP，因此需要直接初始化前后端分离架构。

## 产品分析

### 目标用户与场景
- **用户群体:** 平台管理员、采购专员、销售专员、库管人员
- **使用场景:** 采购单录入、库存监控、销售出库、基础资料维护、经营报表查看
- **核心痛点:** 传统台账分散、流转追踪困难、库存与销售协同弱、系统技术方案需要从 JSP 调整为 Vue

### 价值主张与成功指标
- **价值主张:** 构建可持续扩展的采销存协同平台基础工程，降低后续模块开发成本
- **成功指标:** 仓库具备清晰的前后端目录、后端可启动、前端可构建、核心业务页面与接口具备占位能力

### 人文关怀
系统当前仅初始化工程结构，不落地真实生产数据；后续涉及用户、订单与库存信息时，应遵循最小必要原则处理业务数据，并避免在仓库中存放敏感凭据。

## 变更内容
1. 建立 Spring Boot 后端骨架与基础接口
2. 建立 Vue 前端骨架与业务导航页面
3. 建立 HelloAGENTS 知识库与初始化方案包
4. 更新项目说明，明确使用 Vue 取代 JSP 的前后端分离方案

## 影响范围
- **模块:** backend, frontend, helloagents/wiki, helloagents/history
- **文件:** 根目录说明、Maven 配置、Vue 配置、知识库文档
- **API:** 初始化健康检查、概览数据接口
- **数据:** 文档级数据模型定义，无生产数据落库

## 核心场景

### 需求: 初始化前后端工程
**模块:** 工程结构
根据文档建立可继续开发的统一项目骨架。

#### 场景: 创建前后端目录
仓库当前没有实际代码。
- 预期结果: 生成 backend 与 frontend 两个独立子项目
- 预期结果: 后续开发者可以按模块继续扩展

### 需求: 使用 Vue 替代 JSP
**模块:** 前端
前端技术需明确为 Vue，而不是服务端 JSP 页面。

#### 场景: 建立 Vue 单页应用
用户明确要求前端采用 Vue。
- 预期结果: 前端目录采用 Vue + Vite 结构
- 预期结果: 页面通过路由承载采购、库存、销售、管理等模块

### 需求: 覆盖毕业设计核心模块
**模块:** 业务骨架
需围绕任务书定义的角色职责提供后续扩展入口。

#### 场景: 提供模块占位与接口示例
系统暂未进入完整业务实现阶段。
- 预期结果: 后端提供概览接口与业务模块分层目录
- 预期结果: 前端具备仪表盘与核心菜单页面

## 风险评估
- **风险:** 开题报告 .doc 在当前环境下无法完整抽取，可能遗漏个别细节
- **缓解:** 先以任务书与用户明确要求为准初始化骨架，后续若用户补充开题报告细节，可增量调整
