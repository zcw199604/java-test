# 变更提案: 任务书对齐的后端功能补齐与架构重构

## 需求背景
当前后端已具备登录、基础资料、采购、库存、销售、报表的最小可运行闭环，但与任务书要求存在明显差距：
1. 认证与权限仍采用 JWT + 自定义拦截器，未满足 Apache Shiro + RBAC 要求。
2. 缺少忘记密码/重置密码、验证码、系统配置、日志审计、消息通知、异常审核、合规追溯、Excel 批量导入导出等关键后端能力。
3. 采购、销售、库存、统计模块仅覆盖演示型流程，未形成任务书要求的完整业务闭环。
4. 数据模型与接口设计偏简化，无法支撑审核、取消、追溯、预警、消息与审计等场景。

## 产品分析

### 目标用户与场景
- **用户群体:** 超级管理员、普通管理员（采购专员、销售专员、库管人员）
- **使用场景:** 日常登录鉴权、账号授权、基础资料维护、采购提报/下单/入库、销售发布/下单/出库/回款、库存盘点/调拨/预警、全流程统计追溯与异常审核
- **核心痛点:** 角色能力不完整、审批与审计链缺失、Excel 批处理不足、无法按任务书进行完整验收

### 价值主张与成功指标
- **价值主张:** 将当前演示型后端升级为可按任务书说明进行系统验收的完整业务后端
- **成功指标:**
  - 功能清单覆盖率达到 100%
  - 认证与权限切换为 Apache Shiro + RBAC
  - Excel 导入导出改为 Apache POI 实现
  - 核心业务流具备操作日志、审核状态与追溯链
  - `mvn test` 覆盖核心业务闭环并通过

### 人文关怀
- 密码重置、验证码、操作审计需要兼顾账号安全与最小权限原则。
- 日志与追溯设计避免暴露敏感信息，仅保留必要审计字段。
- 响应式适配主要属于前端能力，本次以后端接口兼容移动端调用为目标，不额外引入终端绑定限制。

## 变更内容
1. 将认证与权限体系由 JWT + 拦截器重构为 Apache Shiro + RBAC。
2. 补齐认证、账号、权限、日志、系统配置、个人中心、通知消息等管理能力。
3. 扩展采购、销售、库存、统计与追溯的数据模型、状态机与接口。
4. 引入 Apache POI，提供 Excel 批量导入导出接口。
5. 建立异常单据、预警通知、合规追溯、审计日志等横切能力。
6. 更新知识库、测试与历史记录，确保代码与文档一致。

- **数据:** 用户、角色、权限、数据范围、日志、通知、验证码、密码重置、系统配置、采购/销售/库存/审核/追溯相关新表与字段

## 核心场景

### 需求: Apache Shiro 认证与 RBAC 权限体系
**模块:** auth
将登录、鉴权、角色、权限点、数据范围统一迁移为 Shiro 驱动，并支持超级管理员与普通管理员差异化授权。

#### 场景: 多角色安全登录
用户输入账号、密码、验证码后完成登录。
- 返回基于 Shiro 会话/令牌的登录结果
- 根据角色与权限点动态返回菜单和能力列表
- 登录失败、验证码错误、账号禁用均有明确错误提示

#### 场景: 忘记密码与重置密码
用户触发忘记密码流程并完成密码重置。
- 验证验证码与账号身份
- 使用加密密码存储与更新
- 写入安全日志与操作日志

### 需求: 超级管理员全量后台管理
**模块:** system
补齐平台管理员能力，包括权限、账号、基础信息、系统配置、日志与个人密码管理。

#### 场景: 管理员维护账号与权限
超级管理员创建、修改、禁用账号并分配角色与数据范围。
- 支持账号增删改查、状态变更、数据范围配置
- 支持角色、权限点、菜单关系维护
- 变更过程写入操作日志

#### 场景: 管理员查看系统日志与配置
超级管理员查看登录日志、操作日志、预警阈值与缓存配置。
- 支持分页查询、筛选与详情查看
- 支持修改系统配置并即时生效或按约定生效

### 需求: 普通管理员业务入口与个人中心
**模块:** system
普通管理员具备个人中心、动态业务入口与基础密码维护能力。

#### 场景: 个人中心维护
普通管理员查看并更新个人资料。
- 支持查看个人信息
- 支持修改密码、头像/联系方式等基础信息（如当前模型允许）
- 权限受角色与数据范围限制

### 需求: 采购管理完整化
**模块:** purchase
补齐采购需求提报、采购单审核/修改/取消、采购跟踪、Excel 导入导出与采购分析。

#### 场景: 采购需求提报到采购入库
采购专员从需求提报开始生成采购单，经过审核、到货、入库完成业务闭环。
- 支持需求单、采购单、状态流转、审核意见
- 支持 Excel 导入导出采购单
- 支持按供应商、品类、时间维度统计采购数据

### 需求: 销售管理完整化
**模块:** sales
补齐销售信息发布、销售单审核/修改/取消、出库、回款状态与销售统计。

#### 场景: 销售发布到回款跟踪
销售专员发布销售信息，创建销售单，经审核、出库、回款形成闭环。
- 支持销售发布信息维护
- 支持销售单审核、取消、修改
- 支持回款状态查询、欠款跟踪与业绩统计

### 需求: 库存管理增强
**模块:** inventory
补齐 Excel 批量维护、出入库/调拨明细、盘点盈亏、自动预警与台账查询。

#### 场景: 库存预警与盘点追踪
库管人员维护库存并处理预警和盘点差异。
- 支持 Excel 批量导入导出库存
- 自动触发库存预警并生成通知
- 支持盘点盈亏上报与历史明细查询

### 需求: 统计分析与合规追溯
**模块:** report
补齐采销存汇总、全流程追溯、异常审核与联动图表。

#### 场景: 单据追溯与异常审核
管理员查看单据在采购、库存、销售环节的完整链路。
- 记录单据状态、操作人、时间、意见与关联流水
- 支持异常单据筛选、审核、监控与报表展示

### 需求: 辅助功能与通知
**模块:** message
提供 Excel 导入导出、操作日志查看、消息通知与预警弹窗所需后端能力。

#### 场景: 预警消息推送
系统检测库存不足或异常单据后生成消息。
- 支持站内消息列表、已读未读管理
- 支持预警类型与业务来源标记

## 风险评估
- **风险:** 改动涉及认证、数据库、业务状态机和接口契约，范围大、联动强。
- **缓解:** 采用“先数据模型与鉴权骨架、再业务模块分阶段迁移、最后统一回归”的策略；所有关键流增加集成测试；保留阶段性兼容适配层，避免一次性失控重构。
